# Gaps between our search and the real engine (`@pkmn/sim`)

The measured, ground-truth list of where our fast lookahead (`endgameSearch.ts`)
disagrees with the actual Pokémon Showdown battle engine. Generated by the diff-harness
(`packages/core/src/domain/simDiff.ts`, run via `npx tsx
packages/core/src/scripts/sim-diff-report.ts [N]`). See also the per-mechanic coverage
in [`mechanics-coverage.md`](mechanics-coverage.md); this file is the *empirical* view.

## How it's measured

For each random doubles position the harness feeds BOTH engines the **same position and
the same concrete move** (the move our cell picked), then compares the **structural
delta** the turn produced — faint / status / boost / weather / terrain. It aligns both
engines to the sim's post-send-out baseline (lead Intimidate/weather) and uses 8 seeds
for the faint check (only flags a faint when the sim is unanimous 0/8 or 8/8 AND we
disagree), so roll-boundary noise is filtered out. Damage-roll differences within the
envelope are expected and ignored — only discrete, roll-independent disagreements count.

## Latest run (2026-06-01, N=400 → 277 valid positions)

**43/277 positions diverged.** By field:

| field | count | cause | verdict |
|---|---|---|---|
| `status` | 15 | 10–30% burn / poison secondaries, Flame Body contact-burn | **policy** — probabilistic, deliberately not auto-applied (like flinch / 25% para) |
| `fainted` | 10 | recoil + incoming damage at the KO boundary (e.g. Talonflame Brave Bird) | **real-ish** — our point-estimate is off by enough that every seed agrees; candidate for a KO-envelope / per-move-cell refinement |
| `boost:spa/def/spd/atk` | 23 | 20% Crunch/Liquidation def-drops, Defiant downstream of unmodelled triggers | **mostly policy** — probabilistic secondaries; a few are real (Defiant reacting to a drop the harness's "attack" action didn't drive) |

## How to read this

- **`status` and the low-% `boost:*` rows are NOT gaps to "fix".** They are
  probabilistic secondaries (a 30% burn, a 20% Def drop). Applying them deterministically
  would be *wrong* (it would overstate them). By design we treat sub-100% secondaries as
  risks, not certainties — the same rule as flinch and the 25% full-paralysis chance. The
  100%-guaranteed drops (Icy Wind/Snarl/Draco self-drop/Parting Shot) ARE modelled and do
  not appear here.
- **`fainted` (~10) is the one genuine residual.** It is recoil + chip at a KO threshold
  where our representative-mid damage disagrees with the sim across *all* seeds. The fix
  is a more precise damage point (per-move cells / a KO-probability envelope), not a new
  mechanic. Tracked as the remaining harness item.

## Harness blind spot (important)

The harness drives **"attack with the best-damage move"** actions only. It therefore does
NOT exercise the action-CHOICE mechanics — sleep, redirection, pivot, Taunt, Encore,
setup, status moves, hazards, recovery, switches. Those are validated by **unit tests**
(`endgame-search.test.ts`) and surfaced by the **`unmodeled.ts` detector**, not by this
divergence count. So "43/277" measures *resolution* fidelity for a chosen attack, not
*action-offering* fidelity. Both are needed; this file covers the former.
